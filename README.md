# Market

Веб сервис поиска товаров на маркетплейсах (Kaspi, Wildberries, Ozon, AliExpress, Alibaba). Пользователь вводит название товара или загружает фото, сервис ищет предложения на нескольких маркетплейсах, приводит их к единому виду, оценивает по цене и качеству и показывает ранжированный список с ссылками на покупку. Интерфейс на трех языках: русский (по умолчанию), английский, казахский.

## Живой адрес

Добавьте сюда адрес после деплоя на Vercel.

## Стек

- React + TypeScript (strict) + Vite + Tailwind CSS v4
- Supabase: Postgres, Auth, Edge Functions на Deno
- Шрифты Rubik (заголовки) и Inter (текст) через @fontsource-variable, полная поддержка казахских букв
- Иконки только из lucide-react
- Деплой: фронтенд на Vercel, Edge Functions на Supabase

## Структура папок

- `src/landing` — лендинг
- `src/app` — основное приложение (поиск, авторизация, избранное, история, детали товара)
- `src/shared` — общий код: i18n, UI компоненты, авторизация, избранное, клиент Supabase, типы
- `supabase/functions` — Edge Functions (`search`, `describe-image`) и общий серверный код с адаптерами маркетплейсов
- `supabase/migrations` — SQL миграции

## Переменные окружения

Скопируйте `.env.example` в `.env` и заполните (в git попадает только `.env.example`):

- `VITE_SUPABASE_URL` — адрес проекта Supabase
- `VITE_SUPABASE_ANON_KEY` — публичный anon ключ

Серверные секреты хранятся в Supabase, а не в коде:

- `VISION_API_KEY` — ключ модели зрения для поиска по фото. Опционально: без него текстовый поиск работает как обычно.

## Как запустить локально

```bash
npm install
cp .env.example .env   # заполните значения
npm run dev
```

## Как развернуть

1. Примените миграции из `supabase/migrations` в SQL Editor проекта Supabase по порядку: `0001`, `0002`, `0003`.
2. Установите Supabase CLI и разверните функции:
   ```bash
   supabase login
   supabase link --project-ref <ваш ref>
   supabase functions deploy search
   supabase functions deploy describe-image
   ```
3. Опционально, для поиска по фото задайте секрет:
   ```bash
   supabase secrets set VISION_API_KEY=<ключ>
   ```
4. Задеплойте фронтенд на Vercel с пресетом Vite, добавьте переменные `VITE_SUPABASE_URL` и `VITE_SUPABASE_ANON_KEY`, затем добавьте адрес Vercel в разрешенные URL в настройках Supabase Auth.

## Оценка (индекс выгоды)

`valueScore = 0.6 * качество + 0.4 * цена`. Качество учитывает рейтинг и число отзывов, цена сравнивается с самым дешевым предложением среди найденных. Результаты сортируются по индексу выгоды по убыванию.

## Известные ограничения по маркетплейсам

Все запросы к маркетплейсам идут через серверные Edge Functions: в браузере это невозможно из за CORS, а токены и обход анти бот защиты не должны попадать во фронтенд.

- **Wildberries** — адаптер обращается к публичному поиску, наиболее вероятно работает.
- **AliExpress, Alibaba** — адаптеры реализованы, но их публичные эндпоинты часто блокируют серверные запросы из дата центров.
- **Ozon** — отключен миграцией `0002`: сильная анти бот защита, блокирует запросы из дата центров.
- **Kaspi** — отключен миграцией `0003`: очень сильная защита. Чтобы включить, нужен официальный API продавца Kaspi, платный прокси или отдельный воркер на резидентном IP.

Заблокированный маркетплейс не ломает поиск: он помечается как временно недоступный, остальные продолжают работать. Адаптеры отключенных маркетплейсов остаются в коде и включаются обратно обновлением строки в таблице `marketplaces`.
